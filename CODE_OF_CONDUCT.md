# Code of Conduct

## Our Standards

Contributors to Hermit KMS are expected to:

- Be respectful and constructive in all project spaces (issues, PRs, discussions)
- Focus feedback on ideas and code, not on individuals
- Accept differing viewpoints and experience levels
- Prioritize the health of the project and its community

## Unacceptable Behavior

The following will not be tolerated:

- Personal attacks or demeaning language directed at any contributor
- Publishing others' private information without consent
- Any conduct that a reasonable person would consider unprofessional

## Enforcement

Instances of unacceptable behavior may be reported by opening a **private** GitHub security advisory or emailing the maintainers directly (see `SECURITY.md` for contact information). Reports will be reviewed promptly and kept confidential.

Maintainers reserve the right to remove, edit, or reject contributions that violate this code of conduct, and to temporarily or permanently ban contributors for repeated or severe violations.

## Scope

This code of conduct applies to all project spaces including the GitHub repository, issue tracker, pull requests, and any official communication channels.
